Created 2020-07-28| Updated 2020-07-29

  公司的统一登录界面需要版本迭代, 原来就是就是html、js、css组成的静态页面, 源码一眼可见, 安全系数很低, 因为项目时间充裕，就打算重写这个统一登录项目来着, 由于拢共就两个页面, 页面交互也不频繁。所以就没选择react和vue的脚手架, 还是采用原生开发, 但是会用gulp来构建

什么是gulp？

  gulp作为一个自动化构建工具, 非常适合构建小型项目, 和webpack相比较, 它的操作更加简便, 采用链式的api很像jquery，只不过jquery的函数执行完后会返回一个jquery对象, 而gulp的每步大概率会返回一个node的stream, 这个下面还会提到

gulp的基本流程

  gulp中有任务(task)的概念, 所有的任务既可以串联执行(series)也可以并行执行(parallel),假如是串联执行的话, 就会有顺序这一说, 就好像一条汽车生产流水线, 你首先得要从焊接车间->涂装车间->装配车间,你搞错一个环节都可能导致程序跑不起来, 最简单的示例如下

搭建脚手架

  在es6之前, 前端开发引入文件的方式只能通过相对或者绝对路径来引用, 没有包管理系统的情况下，去要梳理各个文件之间的引用关系十分困难, 但自从npm问世后, 前端就可以像后端一样进行模块化开发, 所以我们首先要引入一个包管理的库, 来梳理各个页面模块之间的关系, gulp自身只是个流程构建, 所以首先需要引入一个包管理工具,我选择的是browserify
