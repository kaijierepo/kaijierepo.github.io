proxy 与 defineProperty 的异同

  Proxy 对象用于创建一个对象的代理，从而实现基本操作的拦截和自定义（如属性查找、赋值、枚举、函数调用等), 一提到 proxy 就不能不提到 Object.defineProperty()这个 api, 因为它们在一些方面很类似, 均可以劫持对象, 这是 MVVM 框架得以实现的根本, 但均不能在 IE8 及以下运行, 因为内置 proxy 对象根本没有, Object.defineProperty()只能用于 dom 节点不适用于普通对象， 而且这两个 api 还不能在 ie 上进行 pollyfill, 还有就是这两个 api 对于一个深度嵌套的对象都是用递归的方式来监听的。
  说到差异也不少, 要不然 vue 从 2.x 升级到 3.x 也不会用 proxy 代替, 最大的区别就是 defineProperty 不能直接监听数组的变化, 在 js 中数组也是一种特殊的对象, 但是 defineProperty 有个参数是 key, 但是数组没有显式的 key 值，所以 vue 只能间接的监听 Array 对象的方法比如 push, shift, pop 等，所以你不能直接给数组角标去赋值，因为监听不到, 另外 defineProperty 它还不能监听对象属性的添加与删除, 所以才有了$set,$delete 这些方法来弥补, 但是 proxy 的出现这一些缺陷都可以迎刃而解。
